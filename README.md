# Using Machine Learning and Self-Reported Health Data to Classify Diabetes Status

This repository contains the code, outputs, and figures used for the Master's thesis:

**Using Machine Learning and Self-Reported Health Data to Classify Diabetes Status: Evidence from BRFSS 2023**

**Author:** Luke Rodermond  
**Programme:** MSc Data Science & Society  
**University:** Tilburg University  

## Project overview

This project evaluates whether machine learning models can classify diabetes status using self-reported demographic, behavioural, and health-related variables from the 2023 Behavioral Risk Factor Surveillance System (BRFSS).

The thesis focuses on three main questions:

1. Which machine learning model provides the best classification performance for distinguishing between individuals with and without diabetes?
2. Which variables contribute most strongly to diabetes classification in the best-performing model?
3. Do classification performance and error patterns differ across individual and combined demographic subgroups?

The analysis compares logistic regression, random forest, and XGBoost using a final modeling dataset of 149,882 respondents.

## Data

The original BRFSS 2023 dataset was obtained from the Centers for Disease Control and Prevention:

[https://www.cdc.gov/brfss/annual_data/annual_2023.html](https://www.cdc.gov/brfss/annual_data/annual_2023.html)

The raw BRFSS data file is not included in this repository due to file size. To reproduce the full analysis, download the 2023 SAS XPORT file from the CDC and place it in the appropriate working directory.

Expected raw data file name:

```text
LLCP2023.XPT
```

The final modeling dataset is generated by the preprocessing script and is also not included in this repository, because it can be recreated from the raw BRFSS data.

## Predictors and target variable

The target variable is diabetes status, derived from `DIABETE4`.

The final modeling dataset includes the following predictors:

* age
* sex
* education
* income
* smoking status
* physical activity
* alcohol consumption
* BMI category
* hypertension status
* cholesterol status

The target was recoded into a binary classification task:

* `1` = diabetes
* `0` = no diabetes

Ambiguous, non-informative, pregnancy-related, and prediabetes/borderline target responses were excluded from the binary classification task.

## Scripts

The main scripts are located in the `scripts/` folder.

### `eda_brfss_2023.py`

Performs exploratory data analysis and initial data quality checks on the raw BRFSS file. This includes checking predictor availability, target distribution, missingness, value counts, basic plausibility patterns, and potential leakage-related variables.

### `create_modeling_dataset.py`

Creates the final modeling dataset from the raw BRFSS 2023 data. This script recodes the target variable, recodes the selected predictors, handles missing values through row-wise deletion, and saves the final modeling dataset.

### `modeling_diabetes_mvp.py`

Runs the main modeling pipeline. This includes:

* stratified train-test splitting
* stratified 5-fold cross-validation
* class imbalance handling
* randomized hyperparameter search
* out-of-fold threshold selection
* held-out test evaluation
* confusion matrices
* feature importance
* subgroup and combined subgroup metrics

### `nested_cv_model_comparison.py`

Runs the nested cross-validation robustness check used to assess whether the model ranking was stable across multiple data splits. The script uses outer folds for performance estimation and inner folds for hyperparameter tuning, with threshold selection based on out-of-fold predictions.

### `make_fairness_gap_summary.py`

Creates the fairness-relevant gap summary from the subgroup output. It calculates recall gaps, false negative rate gaps, and false positive rate gaps for individual and combined subgroup comparisons.

### `make_thesis_models.py`

Generates the final figures used in the thesis from the saved output files.

## Modeling approach

Three models were compared:

* Logistic Regression
* Random Forest
* XGBoost

Logistic regression was used as an interpretable baseline. Random forest and XGBoost were included as more flexible ensemble-based models capable of capturing non-linear relationships and interactions.

Class imbalance was handled using algorithm-level methods:

* balanced class weights for logistic regression
* balanced subsample weighting for random forest
* `scale_pos_weight` for XGBoost

Oversampling methods such as SMOTE were not used in the final pipeline.

## Evaluation strategy

Model performance was evaluated using both threshold-independent and threshold-dependent metrics:

* ROC-AUC
* PR-AUC
* precision
* recall
* F1-score
* confusion matrix outcomes
* false positive rate
* false negative rate

Threshold selection was performed using out-of-fold predicted probabilities from the training set. The threshold that maximized F1-score was selected for each model and then applied to the held-out test set.

The held-out test set was not used during hyperparameter tuning, threshold selection, or model selection.

As an additional robustness check, nested cross-validation was used to evaluate whether the model ranking was stable across multiple data splits. Paired comparisons were then used to compare model F1-scores across outer folds.

## Main results

XGBoost achieved the strongest overall performance on the held-out test set, but the difference compared with logistic regression and random forest was small.

Final tuned test-set performance:

| Model               | Threshold | ROC-AUC | PR-AUC | Precision | Recall | F1-score |
| ------------------- | --------: | ------: | -----: | --------: | -----: | -------: |
| Logistic Regression |      0.67 |   0.786 |  0.293 |     0.282 |  0.497 |    0.365 |
| Random Forest       |      0.62 |   0.789 |  0.296 |     0.273 |  0.546 |    0.364 |
| XGBoost             |      0.67 |   0.793 |  0.302 |     0.291 |  0.516 |    0.372 |

The results suggest that self-reported BRFSS variables contain meaningful signal for diabetes classification, but predictive performance remains moderate. The findings indicate that performance is likely constrained more by the limited clinical informativeness of self-reported survey variables than by model choice alone.

## Feature importance

Gain-based feature importance was calculated for the tuned XGBoost model and aggregated back to the original variable level.

The most important predictors were:

* hypertension
* BMI category
* cholesterol status

These variables are plausible health-related predictors, but feature importance scores should not be interpreted as causal effects.

## Subgroup analysis

The tuned XGBoost model was evaluated across individual demographic subgroups:

* sex
* age group
* income group
* education group

Combined subgroup analyses were also conducted for:

* age × education
* sex × income

The subgroup analysis showed that recall and error patterns differed across demographic groups. Combined subgroup analyses showed that performance differences became more pronounced when demographic characteristics were considered jointly.

## Fairness-relevant gap analysis

Fairness-relevant performance gaps were calculated from the subgroup output. The gap summary includes:

* recall gaps
* false negative rate gaps
* false positive rate gaps

The largest gaps appeared in the combined subgroup analyses, especially for age × education and sex × income. These results were used to evaluate whether model errors were evenly distributed across population groups.

## Outputs

## Reproducibility

The analysis uses a global random state of 42 where applicable.

The modeling workflow follows this structure:

1. Load and inspect the raw BRFSS 2023 data.
2. Recode the target and selected predictors.
3. Remove observations with missing values in selected modeling variables.
4. Split the final modeling dataset into stratified training and held-out test sets.
5. Tune hyperparameters using stratified 5-fold cross-validation on the training set.
6. Select model-specific thresholds using out-of-fold training predictions.
7. Evaluate final performance on the held-out test set.
8. Generate feature importance values, subgroup metrics, fairness-gap summaries, and thesis figures.
9. Run nested cross-validation as an additional robustness check for model comparison.
